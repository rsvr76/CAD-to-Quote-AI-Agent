# Step 1 - Uploading the Design (Grandma Version)

---

## What are we actually doing here?

The engineer has a 3D design of a metal part on their computer - like a digital blueprint. They upload this file to our system, similar to how you'd attach a photo to an email.

---

## What kind of files can they upload?

Mainly one type:
- **STEP files** (`.step` / `.stp`) — the real CAD exchange format used in manufacturing

STEP is more like the *real blueprint* of the part (exact surfaces and edges), not just a triangle skin.

That matters because we need real CAD surfaces and topology to detect manufacturing features (holes, pockets, faces). A triangle mesh (like STL) can’t reliably tell us that.

---

## What does the system check?

Before doing anything else, the system makes sure the file isn't broken:

1. **Does it actually contain a shape?** - Not an empty file
2. **Does the shape have surfaces?** - Not just invisible points
3. **Does it have real size?** - Not a flat, zero-volume shape
4. **Is it a real closed solid?** - So volume/weight calculations make sense

Extra guardrails (for robustness):

5. **Is the file insanely huge?** - If it’s too big (for example > 100 MB), the system rejects it so the CAD kernel doesn’t crash.
6. **Does it contain multiple bodies?** - Many STEP files have more than one solid. For hackathon scope, we pick the biggest solid by volume (or we could ask the engineer which body to quote).
7. **What units is it in?** - STEP files might be in mm, cm, meters, or inches. We assume mm by default and normalize units in the next step when possible.

If the file passes all checks, we move to Step 2. If not, we tell the engineer what's wrong and ask them to upload a fixed file.

---

## Why is this step needed?

Garbage in = garbage out. If the 3D file is broken, every measurement we extract from it would be wrong, and the cost quote would be meaningless. Better to catch problems at the door.

---

## What does this step produce?

A verified, clean 3D model we can safely measure in the next step.

If the STEP file contains multiple solids, we pick the biggest solid (by volume) for the quote and keep a note that multiple bodies were present.
