# Step 12 - Design Improvement Suggestions (Grandma Version)

---

## What are we actually doing here?

After showing the quote, the system asks: "Could you make this part cheaper by changing the design slightly?" It tests different design changes and ranks them by how much money they'd save.

Think of it like a financial advisor saying: "If you switch from premium fuel to regular, you'd save ₹500/month. If you drive less on weekends, you'd save ₹300/month."

---

## How does it work?

For each possible change:

1. **Tweak one thing** about the part (e.g., fewer holes)
2. **Re-run the AI** with the modified design
3. **Compare costs** - old vs new
4. **Show the savings**

Important: at first, the system is only **giving advice**.

- It does **not** change your design automatically.
- It only changes anything **when you touch/click a suggestion**.

One important realism note (judge-friendly): the system does **not** edit the STEP CAD file by itself. It simulates the change at the “assumptions/features” level to estimate the cost impact, and the engineer applies the change in CAD if they accept it.

---

## What kind of suggestions?

| Suggestion | Why it saves money |
|---|---|
| Relax tolerance (Fine to Standard) | Removes the grinding step entirely |
| Reduce hole count | Less drilling time |
| Simplify pockets | Fewer milling passes |
| Switch to cheaper material | Both material AND machining cost drop |

---

## Example output

```
DFM Suggestions (ranked by savings):

1. Relax tolerance Fine -> Standard : Save ₹2,400
   Reason: removes grinding finishing pass

2. Reduce holes from 6 to 4       : Save ₹560
   Reason: reduces drilling time

3. Use Aluminium instead of Steel  : Save ₹340
   Reason: softer material = faster machining + cheaper raw material
```

---

## What happens when the engineer touches a suggestion?

When the engineer taps a suggestion, the system:

1. Creates a **what-if quote variant** (keeps the original design and quote safe)
2. Applies that one change as a *simulated assumption change* (like “tolerance Standard” or “holes 6 → 4”)
3. Recalculates the new quote
4. Shows the updated savings

Because the engineer explicitly chose to apply a change, the system tags it as an **Engineer override**.

---

## Can we undo everything?

Yes. There is a **Revert all changes** button that brings the design and the quote back to the original version.

Revert clears the applied change and removes the **Engineer override** state for DFM.

---

## Can we see both designs clearly?

Yes. The UI can show two versions **side by side**:

- Original design + original cost
- Modified (simulated) assumptions + new cost

It also clearly lists what changed (like “holes 6 → 4” or “tolerance Fine → Standard”).

---

## Why is this useful?

Most engineers design for function first and worry about cost later. These suggestions let them make informed tradeoffs BEFORE sending the part to manufacturing. Saving ₹2,400 by relaxing a tolerance that doesn't actually matter is free money.

---

## What does this step produce?

A ranked list of "what-if" suggestions with rupee amounts, shown below the quote so the engineer can consider design changes before placing the order.
