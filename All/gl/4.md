# Step 4 - Deciding the Processes (and Choosing Machines) (Grandma Version)

---

## What are we actually doing here?

Based on what we know about the part, the system decides which **manufacturing processes** are needed (turning, milling, drilling, grinding/finishing). Then, for each needed process, it shows which **machines** can do it.

Think of it like a chef:

- First deciding the *steps* (chop, bake, blend)
- Then choosing the *tools* (knife, oven, blender)

---

## What processes can be selected?

| Process | When it's needed | Simple explanation |
|---|---|---|
| Turning (lathe) | Part is long and thin | Like spinning a piece of wood on a pottery wheel and shaping it |
| Milling | Part has pockets or large surfaces | Like a robot drill that carves shapes from a block |
| Drilling | Part has holes | Exactly what it sounds like - making holes |
| Grinding | Part needs to be very precise | Like sanding but extremely fine - gets the surface super smooth |

---

## How does the system decide?

Simple rules, not AI:

- **Long and thin part?** -> Needs turning
- **Has pockets or big surface?** -> Needs milling
- **Has any holes?** -> Needs drilling
- **Needs high precision?** -> Needs grinding after everything else

In our hackathon version, “grinding” is a simple way to represent a **precision finishing operation** needed for tight tolerances.

---

## Why use simple rules instead of AI here?

Because this decision is straightforward. A manufacturing engineer would make the same decision in 2 seconds - no AI needed. Using AI for simple yes/no decisions would be overcomplicating things.

---

## What does the system show?

```
Processes identified:
  Turning    - OFF (part is not elongated)
  Milling    - ON  (2 pockets detected)
  Drilling   - ON  (6 holes)
  Grinding   - ON  (tolerance = Fine)
```

The engineer can see WHY each machine was selected, making the decision transparent.

---

## Can the engineer change it?

Yes. The dashboard shows the system’s suggested processes **with a simple reason**, and the engineer can switch any of them **ON or OFF** if they know something the system can’t see (special tooling, a different shop route, etc.).

## What machines can do each process?

For every active process, the UI also shows a small dropdown of **machines that can do that process** (for example: 3‑Axis vs 5‑Axis milling, CNC lathe vs manual lathe).

When the engineer changes the machine, the system updates the **hourly rate** for that machine and the quote updates.

When they change it, the system remembers it as an **Engineer override** (optionally with a short note like “use 5‑axis for pocket access”).

If the engineer makes a choice that looks contradictory (for example: the part has holes but drilling is turned OFF), the system shows a warning and asks for a short note explaining why (like “holes are pre-drilled” or “handled manually”).

---

## What does this step produce?

Four yes/no flags (turning, milling, drilling, grinding) that tell the system which machining processes this part will use.

The system starts with an automatic suggestion, but it uses the **final, engineer-confirmed** choices for the cost prediction.

Important: the AI time prediction uses the **final confirmed** process flags (after the engineer’s confirmation), not the automatic suggestion.
