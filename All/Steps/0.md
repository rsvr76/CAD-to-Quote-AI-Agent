# Step 0 - AI Agent Receives User Request

**Phase:** 0 - Agent Initialization
**Input:** User uploads CAD file or initiates quoting request
**Output:** Agent goal established, pipeline orchestration begins

---

## Goal

The AI Quote Agent receives the user's request and orchestrates the entire quoting pipeline. This is the entry point where the system transitions from a passive tool to an active manufacturing assistant.

Manufacturing quoting requires coordinating multiple specialized modules (CAD analysis, process planning, ML time prediction, cost modeling, explainability, and DFM). The agent provides a single orchestration layer that manages this workflow and interacts with the engineer when extra information is required.

---

## What the Agent Does

The agent orchestrates a mostly predefined workflow:

```
Understand → Analyze → Predict → Calculate → Explain → Recommend
```

### 1. Understand the Task

The user uploads a CAD file. The agent establishes its goal:

```
Goal: Generate a transparent manufacturing quote for the uploaded part.
```

### 2. Plan the Tool Chain

The agent orchestrates a predefined tool chain designed for manufacturing quoting (with a few conditional branches for missing inputs, validation failures, and low-confidence outputs):

```python
TOOL_CHAIN = [
    "cad_analyzer",          # Step 1-2: Load and extract geometry
    "input_collector",       # Step 3: Gather missing manufacturing inputs
    "routing_engine",        # Step 4: Determine required processes
    "stock_estimator",       # Step 5-6: Estimate and confirm stock weight
    "feature_assembler",     # Step 7: Build ML feature vector
    "machining_predictor",   # Step 8: Predict machining time (RF)
    "interval_calculator",   # Step 9: Compute uncertainty range
    "cost_engine",           # Step 10: Calculate all cost components
    "explainability_module", # Step 11: Generate SHAP + formula + routing explanations
    "dfm_advisor",           # Step 12: Suggest design improvements
    "quote_presenter",       # Step 13: Assemble and display final quote
]
```

### 3. Execute Tools Sequentially

The agent calls each tool, passing outputs from one as inputs to the next. If any tool requires human input (Steps 3, 6), the agent pauses and engages the engineer through conversation.

### 4. Handle Edge Cases

The agent manages error states:
- Invalid CAD file → request re-upload (Step 1 validation)
- Missing user inputs → ask again with engineering context notes
- Low confidence prediction → flag to engineer with explanation

## Agent Tools

| Tool | Module | Purpose |
|---|---|---|
| CAD Analyzer | OpenCascade (pythonocc-core) | Parse STEP B-Rep and extract geometry + manufacturing features |
| Process Routing Engine | Rules | Determine required machining processes |
| ML Machining Predictor | RandomForestRegressor | Predict machining time |
| Cost Engine | Formulas | Calculate material, setup, overhead costs |
| Explainability Module | SHAP + formulas | Explain cost drivers |
| DFM Advisor | Model re-prediction | Suggest design improvements with savings |

---

## Why This Is an AI Agent

This is best described as an **AI-assisted orchestration layer**: it coordinates multiple modules end-to-end, pauses for human inputs when required, and composes a final output with explanations and recommendations. It is not claiming to be an autonomous planner that invents new tools or policies on the fly.

| Agent Capability | How Our System Does It |
|---|---|
| Understand a task | User uploads CAD, agent sets goal: generate quote |
| Collect information | Chat interface gathers holes, material, tolerance |
| Use tools | Calls CAD parser (OpenCascade), RF model, cost formulas, SHAP |
| Reason about results | Interprets SHAP values, identifies cost drivers |
| Provide recommendations | DFM suggestions with quantified savings |
