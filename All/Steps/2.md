# Step 2 — Geometry Extraction (STEP)

**Phase:** 1 — CAD Processing  
**Input:** Validated CAD model object (from Step 1)  
**Output:** Geometry feature table

---

## Goal

Using an OpenCascade CAD kernel (pythonocc-core), extract measurable geometry properties from the uploaded STEP B-Rep. These properties feed into process routing, stock estimation, and the ML feature vector.

These geometric descriptors provide compact numerical signals that correlate with machining effort and are used as inputs to the Random Forest machining time predictor.

---

## Features Extracted

| Feature | Formula / Method | Purpose |
|---|---|---|
| `volume_cm3` | B-Rep volume properties | Material estimation |
| `surface_area_cm2` | B-Rep surface properties | Machining effort indicator |
| `bbox_x`, `bbox_y`, `bbox_z` | CAD bounding box | Stock estimation |
| `aspect_ratio` | `max(dims) / min(dims)` | Turning detection |
| `sv_ratio` | `surface_area / volume` | Complexity — thin-wall vs solid |
| `slenderness` | `max(dims) / cbrt(volume_mm3)` | Elongation proxy (dimensionless) |
| `num_faces` | Count B-Rep faces | Complexity proxy |
| `num_edges` | Count B-Rep edges | Complexity proxy |
| `pocket_density_proxy` | Computed in Step 7 from **confirmed** pocket count | Bounded pocket-density proxy |

In STEP mode, we also generate *defaults* for the chat step (Step 3):

Optionally, we also compute a simple confidence label (low/medium/high) for each detected estimate so the UI can guide the engineer on what to double-check.

| Feature | Method | How it’s used |
|---|---|---|
| `num_holes_est` | Detect cylindrical faces (hole candidates) | Pre-fill hole count for engineer confirmation |
| `num_pockets_est` | Detect concave regions / pocket candidates | Pre-fill pocket count for engineer confirmation |
| `max_depth_mm_est` | Estimate from hole candidate depth | Pre-fill drilling depth for confirmation |

## Implementation

```python
import numpy as np
from OCC.Core.BRepGProp import brepgprop_VolumeProperties, brepgprop_SurfaceProperties
from OCC.Core.GProp import GProp_GProps
from OCC.Core.Bnd import Bnd_Box
from OCC.Core.BRepBndLib import brepbndlib_Add
from OCC.Core.TopExp import TopExp_Explorer
from OCC.Core.TopAbs import TopAbs_FACE, TopAbs_EDGE

def _count_topology(shape) -> tuple[int, int]:
    faces = 0
    edges = 0
    exp_f = TopExp_Explorer(shape, TopAbs_FACE)
    while exp_f.More():
        faces += 1
        exp_f.Next()
    exp_e = TopExp_Explorer(shape, TopAbs_EDGE)
    while exp_e.More():
        edges += 1
        exp_e.Next()
    return faces, edges

def extract_geometry_from_cad_model(cad_model: dict) -> dict:
    """Extract geometry features from a validated STEP B-Rep shape.

    Notes on units:
    - STEP units can be mm / cm / m / inch. For hackathon scope, we assume **mm** by default.
    - OpenCascade returns volume in (unit^3) and area in (unit^2).
    - If units are mm, then mm^3 and mm^2.
    - We convert mm^3 → cm^3 by dividing by 1000, and mm^2 → cm^2 by dividing by 100.
    - Unit normalization can be enforced when metadata is available.
    """
    shape = cad_model["shape"]

    # Volume
    vprops = GProp_GProps()
    brepgprop_VolumeProperties(shape, vprops)
    volume_mm3 = float(vprops.Mass())
    volume_cm3 = volume_mm3 / 1000.0

    # Surface area
    aprops = GProp_GProps()
    brepgprop_SurfaceProperties(shape, aprops)
    area_mm2 = float(aprops.Mass())
    surface_area_cm2 = area_mm2 / 100.0

    # Bounding box (axis-aligned). Note: may include tolerance padding.
    # For higher precision, an oriented bounding box (OBB) can be computed.
    box = Bnd_Box()
    box.SetGap(0.0)
    brepbndlib_Add(shape, box, True)
    xmin, ymin, zmin, xmax, ymax, zmax = box.Get()
    bbox_x = float(xmax - xmin)
    bbox_y = float(ymax - ymin)
    bbox_z = float(zmax - zmin)
    dims = [bbox_x, bbox_y, bbox_z]

    aspect_ratio = max(dims) / max(min(dims), 1e-6)
    sv_ratio = surface_area_cm2 / max(volume_cm3, 1e-6)

    # Slenderness proxy: compare the longest dimension to a characteristic length scale.
    # This stays dimensionless (if consistent units are used for both terms).
    characteristic_length_mm = max(volume_mm3 ** (1/3), 1e-6)
    slenderness = max(dims) / characteristic_length_mm

    num_faces, num_edges = _count_topology(shape)

    return {
        "volume_cm3": round(volume_cm3, 2),
        "surface_area_cm2": round(surface_area_cm2, 2),
        "bbox_x": round(bbox_x, 2),
        "bbox_y": round(bbox_y, 2),
        "bbox_z": round(bbox_z, 2),
        "aspect_ratio": round(aspect_ratio, 2),
        "sv_ratio": round(sv_ratio, 4),
        "slenderness": round(slenderness, 4),
        "num_faces": int(num_faces),
        "num_edges": int(num_edges),
    }
```

## STEP Notes (Why This Is Better Than STL)

Compared to STL meshes, STEP B-Rep gives exact geometry. That improves:

- Volume and surface area accuracy (not triangle-resolution dependent)
- Bounding box correctness (less sensitive to mesh defects)

Manufacturing feature detection (holes/pockets/depth) becomes possible from CAD topology.
We still confirm with the engineer in Step 3 to avoid edge cases (counterbores, slots, partial cylinders).

---

## Topology Feature Detection (High-Level)

This is the STEP-only upgrade: instead of guessing from triangles, we traverse CAD faces/edges.

- **Hole candidates:** iterate faces → identify cylindrical surfaces → apply radius/axis-length filters → count
- **Pocket candidates:** detect concave faces / inward loops → group into pocket regions → count
- **Depth estimate:** for each hole candidate, approximate depth along its axis; take max

These are **estimates**, not guarantees — Step 3 keeps the human-in-the-loop confirmation.

These are collected from the engineer in Step 3 (User Inputs via Chat).
