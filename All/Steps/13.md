## Cost Contribution Chart (NEW)

To help engineers see “where the money goes” at a glance, display a donut/pie chart showing the **percentage contribution** of each cost component:

- Material
- Machining
- Setup
- Overhead

Data source: Step 10 output (`material_cost`, `machining_cost`, `setup_cost`, `overhead_cost`, `total_cost`).

Recommended implementation (Streamlit + Plotly):

```python
import plotly.graph_objects as go

def create_cost_contribution_chart(costs: dict):
	labels = ["Material", "Machining", "Setup", "Overhead"]
	values = [
		costs["material_cost"],
		costs["machining_cost"],
		costs["setup_cost"],
		costs["overhead_cost"],
	]
	total_cost = costs["total_cost"]
	fig = go.Figure(data=[go.Pie(
		labels=labels,
		values=values,
		hole=0.50,
		textinfo="label+percent",
		insidetextorientation="radial",
	)])

	# Professional touch: show total cost in the donut center
	fig.update_layout(
		title="Cost Contribution",
		showlegend=True,
		annotations=[
			dict(
				text=f"₹{total_cost:,.0f}<br>Total Cost",
				x=0.5,
				y=0.5,
				showarrow=False,
				font_size=18,
			)
		],
	)
	return fig


def find_largest_cost_driver(costs: dict):
	components = {
		"Material": costs["material_cost"],
		"Machining": costs["machining_cost"],
		"Setup": costs["setup_cost"],
		"Overhead": costs["overhead_cost"],
	}
	largest = max(components, key=components.get)
	return largest, components[largest]


def calculate_percentages(costs: dict):
	total = max(float(costs["total_cost"]), 1.0)
	return {
		"Material": round(costs["material_cost"] / total * 100, 1),
		"Machining": round(costs["machining_cost"] / total * 100, 1),
		"Setup": round(costs["setup_cost"] / total * 100, 1),
		"Overhead": round(costs["overhead_cost"] / total * 100, 1),
	}
```

Recommended UI placement:

1. Breakdown with % labels
2. “Largest Cost Driver: Machining (₹…)” callout
3. Donut chart

## PDF-Ready Quote Summary

Provide a one-click export of the quote summary (inputs + breakdown + range + explanations + DFM suggestions) as a PDF.

If a DFM **quote variant** (simulated assumptions) is active, the export includes an additional option:

- **Comparison PDF (side-by-side)** — prints **Original vs Modified assumptions** with the feature/input deltas and the respective cost breakdowns so higher-ups can approve the tradeoff quickly.

This is not a separate model — it is a formatting/export layer for the same results already computed in Steps 8–12.

---

## DFM Suggestions (below explanations)

Ranked design improvement suggestions with quantified savings (sorted by savings).

The list is advisory by default. When the engineer clicks a suggestion, the system applies it to a **simulated quote variant** (feature/assumption change), recalculates the quote, and shows a **side-by-side** comparison (Original vs Modified assumptions). A **Revert all changes** action restores the original quote.

Applying (or reverting) DFM suggestions is recorded as an **Engineer override** action because it is an explicit engineer-driven change via the UI.
