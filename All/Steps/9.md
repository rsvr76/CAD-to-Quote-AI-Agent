# Step 9 — Prediction Interval

**Phase:** 4 — Prediction  
**Input:** Feature vector (Step 7) + Trained RF model  
**Output:** Machining time uncertainty range (10th–90th percentile)

---

## Goal

Compute a prediction interval for machining time using the individual tree predictions within the Random Forest. This gives the engineer a confidence range, not just a point estimate.

Interpretation note (judge-proof):

> The prediction interval represents model uncertainty derived from disagreement among trees in the Random Forest ensemble. It is not a classical statistical confidence interval.

Implementation phase note:

> During the **backend-first** build (before the trained RF model is integrated), this step is implemented as a **heuristic confidence/risk range** (see `backend/confidence.py` in the implementation plan). Once the RF model is integrated, the interval switches to the tree-disagreement method documented below.

---

## How It Works

Random Forest contains many decision trees (e.g., 300).
Each tree produces its own independent prediction.
The spread of these predictions indicates model uncertainty.

---

## Implementation

```python
import numpy as np
import pandas as pd

def prediction_interval(
    model,
    features: dict,
    *,
    mean_time_min: float,
    feature_order: list[str],
    lower_pct: int = 10,
    upper_pct: int = 90,
    min_minutes: float = 0.5,
    enforce_min_width_ratio: float | None = None,
) -> dict:
    """Compute prediction interval from individual tree predictions.

    - mean_time_min should come from Step 8 (model.predict) to avoid tiny averaging discrepancies.
    - feature_order must match the training schema exactly.
    """

    # Guaranteed column order (never rely on dict ordering)
    row = [features[name] for name in feature_order]
    feature_df = pd.DataFrame([row], columns=feature_order)
    
    # Collect predictions from all trees
    tree_predictions = np.array([
        tree.predict(feature_df)[0] 
        for tree in model.estimators_
    ])

    lower_bound = float(np.percentile(tree_predictions, lower_pct))
    upper_bound = float(np.percentile(tree_predictions, upper_pct))

    # Sanity floors (protect downstream cost math)
    mean_time_min = max(min_minutes, float(mean_time_min))
    lower_bound = max(min_minutes, lower_bound)
    upper_bound = max(min_minutes, upper_bound)

    # Optional: avoid a misleading “zero uncertainty” edge case
    if enforce_min_width_ratio is not None:
        min_width = mean_time_min * enforce_min_width_ratio
        if (upper_bound - lower_bound) < min_width:
            upper_bound = lower_bound + min_width

    interval_width = max(0.0, upper_bound - lower_bound)
    risk_ratio = interval_width / mean_time_min

    if risk_ratio < 0.10:
        risk_level = "Low"
    elif risk_ratio <= 0.25:
        risk_level = "Medium"
    else:
        risk_level = "High"
    
    return {
        "mean_time_min": round(mean_time_min, 2),
        "lower_time_min": round(lower_bound, 2),
        "upper_time_min": round(upper_bound, 2),
        "tree_count": len(tree_predictions),
        "interval_pct": [lower_pct, upper_pct],
        "risk_ratio": round(risk_ratio, 3),
        "risk_level": risk_level,
    }
```


## Converting to Cost Range

The time interval converts to a **cost interval** in Step 10 using the **same costing logic** as the point estimate, including the engineer’s selected machines (Step 4) and their per-machine hourly rates.

---

## Important Constraint

The uncertainty applies **only to machining cost**, because:
- Material cost → deterministic (formula)
- Setup cost → deterministic (formula)
- Overhead → deterministic (rule)

Only machining time is predicted by ML, so only machining cost has uncertainty.

Practical tip: a wider interval typically means the feature combination is less common/less represented in training data, so the model’s trees disagree more.
