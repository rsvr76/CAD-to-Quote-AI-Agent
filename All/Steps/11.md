# Step 11 - Explainability

**Phase:** 5 - Explanation Layer
**Input:** ML prediction (Step 8) + Cost breakdown (Step 10) + Routing (Step 4)
**Output:** Three separate explanation layers

---

## Goal

Generate three distinct explanations. Each layer explains a different aspect of the quote. They are kept separate to avoid incorrect interpretation.

Implementation phase note:

> During the **backend-first** build (before the trained RF + SHAP integration), the UI can show a lightweight **rule-based “top drivers”** list (from deterministic formulas + routing reasons). Once ML is integrated, this layer becomes the full **SHAP per-quote** explanation described below.

---

## Layer 1: ML Explanation (SHAP)

SHAP identifies which features affected machining time for this specific part.

Uses TreeExplainer on the Random Forest model.

Important: SHAP values are in the same units as the model output.

> Here, SHAP values represent contributions to predicted machining time (minutes), not cost. Any cost impact is derived later by converting time to ₹ using the selected machine rates.

Also note:

> Machine selection affects cost but does not change the ML time prediction (the model does not take machine choice as an input).

```python
import shap

# Build the explainer once (startup / module init) and reuse it.
# Creating TreeExplainer can be ~50–100ms, so caching improves responsiveness.
def build_shap_explainer(model):
  return shap.TreeExplainer(model)


def explain_machining_time(explainer, features_df):
  # Preferred modern SHAP API: call the explainer.
  # Returns a shap.Explanation object.
  explanation = explainer(features_df)
  return {
    "base_values": explanation.base_values,
    "values": explanation.values,
    "feature_names": explanation.feature_names,
  }
```

**Example output:**

```
Why does this part take 7.5 min to machine?

  Base (dataset mean)        : 4.2 min
  + Grinding active          : +1.8 min
  + Drilling work            : +2.1 min
  - Aluminium material       : -0.9 min
  + Fine tolerance           : +0.3 min
  ------------------------------------
  = This part's machining    : 7.5 min
```

This is a per-quote waterfall, not global feature importance.

## Why Three Separate Layers

| Layer | What it explains | Method |
|---|---|---|
| SHAP | Why machining TIME changed | ML model introspection |
| Formula | How each COST was calculated | Deterministic math |
| Routing | Why PROCESSES were selected | Rule evaluation |

Mixing these leads to confusion. An engineer asking "why is this expensive?" gets a different answer than "how was this calculated" or "why does this need grinding?"

If someone asks “what feature increased cost?”, the correct interpretation is:

SHAP → time driver (minutes) → Step 10 rate application → ₹ impact.
