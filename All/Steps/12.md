# Step 12 - DFM Suggestions

**Phase:** 6 - Design Feedback
**Input:** Feature vector (Step 7) + Trained model + Cost model (Step 10)
**Output:** Ranked list of design improvement suggestions with savings

---

## Goal

Evaluate potential design improvements by modifying one feature at a time, re-running the model, and measuring the cost difference.

Important UX constraint for this step:

- The system initially **only recommends** changes (ranked by savings).
- The system does **not modify STEP CAD geometry** in hackathon scope.
- On selection, the system creates a **what-if quote variant** (keeps the original), re-quotes it, and shows the difference.

Judge-safe feasibility statement:

> Suggestions are simulated at the feature/requirements level to estimate cost impact. The engineer applies any accepted design change in CAD.

---

## Method

For each suggestion:

1. Modify one feature in the feature vector
2. Re-run the ML model to get new machining_time
3. Recalculate full cost using Step 10 formulas
4. Compare with original cost
5. Rank by savings

Corrected pipeline (ensures consistency):

For each candidate suggestion:

1. Modify the *upstream inputs* (user inputs / requirements), not the final feature dict
2. Re-run routing rules (Step 4) to get `routing_final_variant`
3. Rebuild the full feature vector (Step 7) for the variant
4. Predict time (Step 8) and optionally interval (Step 9)
5. Recompute full cost (Step 10)
6. Compute savings and rank

---

## UI Behavior (Apply-On-Click)

### 1) Suggest (no changes applied)

Show a ranked list sorted by `savings_inr` so the engineer immediately sees “what to do”:

- `Change`
- `Saves (₹)`
- `Reason`

At this stage, these are **what-if previews** only.

### 2) Apply (user taps a suggestion)

When the engineer taps a suggestion, the system:

1. Creates a **quote variant** (keeps the original design + quote unchanged)
2. Applies the corresponding *input assumption change* (e.g., tolerance class change request, “reduce holes” intent)
3. Re-runs routing (Step 4) and rebuilds the feature vector (Step 7) for the variant
4. Marks the suggestion as **Applied**
5. Records the action as an **Engineer override** (since the engineer intentionally changed the design assumptions via the UI)

### 3) Revert

The UI provides a **Revert all changes** action that restores the original design + original quote in one click.

Revert clears the applied suggestions and removes the **Engineer override** state for DFM.

---

## Side-by-Side Compare View

When at least one suggestion is applied, the UI can show a two-column compare:

- **Original**: original design snapshot + original cost breakdown
- **Modified (Simulated)**: same base design snapshot + updated cost breakdown + a clear list of “assumptions changed”

The compare view highlights:

- Applied suggestions list
- Key feature deltas (e.g., holes 6 → 4, tolerance Fine → Standard)
- Total cost delta and the top contributing cost components

## Suggestion Types

### Tolerance Relaxation

```python
def simulate_variant_quote(*, geometry, user_inputs, routing_context, quantity, model, feature_order):
    # 1) Re-run routing from modified inputs
    routing_final_variant, machine_selection_final_variant = route_processes(
        geometry=geometry,
        user_inputs=user_inputs,
        context=routing_context,
    )
    # 2) Rebuild features from sources (no manual patching)
    features_variant, X_row = assemble_features(geometry, user_inputs, routing_final_variant)
    # 3) Predict time + cost
    time_min = predict_machining_time(features_variant)
    quote = calculate_total_quote(
        stock_weight_kg=routing_context["stock_weight_final_kg"],
        machining_time_min=time_min,
        material=user_inputs["material"],
        quantity=quantity,
        routing_final=routing_final_variant,
        machine_selection_final=machine_selection_final_variant,
    )
    return {"features": features_variant, "routing": routing_final_variant, "machines": machine_selection_final_variant, "time_min": time_min, "quote": quote}


def suggest_tolerance_relaxation(geometry, user_inputs, routing_context, original_total_cost):
    if user_inputs["tolerance_class"] in ("Fine", "Ultra-Fine"):
        modified_inputs = user_inputs.copy()
        modified_inputs["tolerance_class"] = "Standard"

        variant = simulate_variant_quote(
            geometry=geometry,
            user_inputs=modified_inputs,
            routing_context=routing_context,
            quantity=routing_context["quantity"],
            model=routing_context["model"],
            feature_order=routing_context["feature_order"],
        )
        savings = original_total_cost - variant["quote"]["total_cost"]
        if savings > 0:
            return {
                "change": "Relax tolerance to Standard",
                "reason": "may remove grinding/finishing requirement",
                "savings_inr": round(savings, 2),
                "variant_inputs": {"tolerance_class": "Standard"},
            }
```

### Hole Count Reduction

```python
def suggest_hole_reduction(geometry, user_inputs, routing_context, original_total_cost):
    holes = user_inputs["num_holes"]
    # Constraint: never suggest absurd drops (hackathon-safe heuristic)
    if holes >= 6:
        modified_inputs = user_inputs.copy()
        modified_inputs["num_holes"] = max(holes - 2, 1)

        variant = simulate_variant_quote(
            geometry=geometry,
            user_inputs=modified_inputs,
            routing_context=routing_context,
            quantity=routing_context["quantity"],
            model=routing_context["model"],
            feature_order=routing_context["feature_order"],
        )
        savings = original_total_cost - variant["quote"]["total_cost"]
        if savings > 0:
            return {
                "change": f"Reduce holes from {holes} to {modified_inputs['num_holes']}",
                "reason": "reduces drilling work and complexity",
                "savings_inr": round(savings, 2),
                "variant_inputs": {"num_holes": modified_inputs["num_holes"]},
            }
```

### Pocket Simplification

Reduce pocket count and recalculate milling impact.

### Material Change

Switch material and recalculate both machining time AND material cost (since material affects both).

## Key Design Decision

Material change suggestions include both:
- Machining cost difference (from re-prediction)
- Material cost difference (from formula)

This ensures total savings are accurate, not just machining savings.
