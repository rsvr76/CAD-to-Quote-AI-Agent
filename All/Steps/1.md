# Step 1 — STEP CAD Upload

**Phase:** 1 — CAD Processing  
**Input:** STEP file from engineer  
**Output:** Validated CAD model object (B-Rep + metadata)

---

## Goal

Engineer uploads a CAD file in STEP format. The system validates the CAD topology before any processing begins.

> STEP format is used because it preserves CAD topology and parametric surfaces required for manufacturing feature detection, which is not possible with triangle mesh formats.

---

## Supported Formats

| Format | Extension |
|---|---|
| STEP | `.step`, `.stp` |

---

## Validation Checks

The uploaded file must pass these checks:

| Check | Condition | Failure Action |
|---|---|---|
| File size guard | File size ≤ configured limit (e.g., 100 MB) | Reject — file too large |
| STEP loads | Reader returns success status | Reject upload, show error |
| Shape exists | Loaded shape is not null | Reject — empty CAD |
| Shape is valid | Topology checker passes | Reject — corrupted geometry |
| Solid count | At least 1 solid found (multi-solid allowed) | Reject — no solid bodies |
| Volume > 0 | Largest solid has positive volume | Reject — non-solid / invalid solid |

---

## Notes (Units)

STEP files may be authored in mm / cm / m / inch. For hackathon scope, we assume **mm** by default and normalize units during Geometry Extraction (Step 2) when metadata is available.

## Implementation

```python
import os

from OCC.Core.STEPControl import STEPControl_Reader
from OCC.Core.IFSelect import IFSelect_RetDone
from OCC.Core.BRepCheck import BRepCheck_Analyzer
from OCC.Core.BRepGProp import brepgprop_VolumeProperties
from OCC.Core.GProp import GProp_GProps
from OCC.Core.TopExp import TopExp_Explorer
from OCC.Core.TopAbs import TopAbs_SOLID

MAX_STEP_BYTES = 100 * 1024 * 1024  # 100 MB

def _iter_solids(shape):
    exp = TopExp_Explorer(shape, TopAbs_SOLID)
    while exp.More():
        yield exp.Current()
        exp.Next()

def _volume(shape) -> float:
    props = GProp_GProps()
    brepgprop_VolumeProperties(shape, props)
    return float(props.Mass())

def load_and_validate_step(filepath: str):
    """Load STEP file and validate B-Rep topology.

    Returns a structured object so downstream steps can handle multi-body files.
    """
    if os.path.getsize(filepath) > MAX_STEP_BYTES:
        raise ValueError("STEP file too large (guardrail)")

    reader = STEPControl_Reader()
    status = reader.ReadFile(filepath)
    if status != IFSelect_RetDone:
        raise ValueError("Failed to read STEP file")

    reader.TransferRoots()
    shape = reader.OneShape()
    if shape.IsNull():
        raise ValueError("STEP loaded but contains no shape")

    analyzer = BRepCheck_Analyzer(shape)
    if not analyzer.IsValid():
        raise ValueError("STEP shape topology is invalid (corrupted geometry)")

    solids = list(_iter_solids(shape))
    if len(solids) == 0:
        raise ValueError("STEP contains no solid bodies")

    # Multi-solid handling: choose the largest solid for quoting (hackathon default).
    # Alternative: quote all solids or ask engineer which body to quote.
    largest_solid = max(solids, key=_volume)
    if _volume(largest_solid) <= 0:
        raise ValueError("STEP solid has zero/negative volume — not a closed solid")

    cad_model = {
        "shape": largest_solid,
        "solid_count": len(solids),
        "unit_system": "mm",  # normalized/confirmed in Step 2 when metadata is available
        "file_size_bytes": os.path.getsize(filepath),
    }
    return cad_model
```

---

## Output

A structured CAD object passed to Step 2:

```python
cad_model = {
    "shape": <TopoDS_Solid>,
    "solid_count": 2,
    "unit_system": "mm",
    "file_size_bytes": 1248392,
}
```

Step 2 uses `cad_model["shape"]` for geometry + feature extraction and uses `cad_model["unit_system"]` for unit normalization.

---

## Error Handling

If validation fails:
- Display specific error message to the engineer
- Allow re-upload without restarting the session
- Log the failure reason for debugging
