# Step 5 — Stock Estimation

**Phase:** 3 — Cost Engine  
**Input:** Geometry features (Step 2) + Routing flags (Step 4) + Material (Step 3)  
**Output:** Estimated and final confirmed stock weight (kg)

---

## Goal

Estimate the raw stock material required to machine the part. This is needed for material cost calculation.

This step produces a best-effort estimate from geometry + routing, and exposes it in the dashboard so the engineer can **override** the stock assumption when needed (standard bar sizes, available off-cuts, supplier constraints).

Important note on hybrid parts:

> When turning is active, the system assumes **cylindrical bar stock** as the starting material even if secondary milling operations are also required (e.g., shafts with milled flats).

---

## Stock Shape Logic

The stock shape depends on whether turning is active:

| Condition | Stock Shape | Reason |
|---|---|---|
| `is_turning == 1` | Cylindrical bar stock | Lathe operations start from round bar |
| `is_turning == 0` | Rectangular block stock | Milling starts from block |

---

## Cylindrical Stock (Turning Active)

```python
import math

def cylindrical_stock_volume(bbox_x, bbox_y, bbox_z):
    """Estimate cylinder stock from bounding box."""
    # Heuristic (hackathon): assume the turning axis aligns with the longest bounding-box dimension.
    # Then use the larger of the remaining two dimensions as the required bar diameter.
    dims = [bbox_x, bbox_y, bbox_z]
    length = max(dims)
    cross_section = [d for d in dims if d != length]
    # Fallback for repeated dimensions
    if len(cross_section) < 2:
        cross_section = sorted(dims)[:2]

    TURNING_DIAMETER_MARGIN = 0.10  # 10% allowance (typical: 5–15%)
    diameter = max(cross_section) * (1.0 + TURNING_DIAMETER_MARGIN)
    
    volume_mm3 = math.pi * (diameter / 2) ** 2 * length
    return volume_mm3 / 1000.0  # convert to cm³
```

## Rectangular Stock (No Turning)

```python
def block_stock_volume(bbox_x, bbox_y, bbox_z):
    """Estimate block stock from bounding box with margins."""
    # Add margin on each side for clamping + tool clearance + roughing allowance.
    # Typical allowances are ~5–10 mm per dimension in many shops.
    MILLING_MARGIN_MM = 5  # per side
    volume_mm3 = (bbox_x + 2 * MILLING_MARGIN_MM) * (bbox_y + 2 * MILLING_MARGIN_MM) * (bbox_z + 2 * MILLING_MARGIN_MM)
    return volume_mm3 / 1000.0  # convert to cm³
```

---

## Notes (Orientation + Scrap)

- **Bounding box orientation:** this estimate assumes the part is roughly aligned with the intended machining setup axes. A rotated model (e.g., 45°) can inflate the axis-aligned bounding box and overestimate stock. The engineer override is the intended correction mechanism in hackathon scope.
- **Scrap recovery:** the hackathon model does not account for scrap resale/reuse value. In production, scrap recovery and standard stock availability catalogs can be integrated.

## Weight Calculation

```python
DENSITY = {
    "Aluminium": 2.70,   # g/cm³
    "Steel":     7.85,
    "Brass":     8.50,
    "Titanium":  4.50,
    "ABS":       1.05,
}

def estimate_stock_weight(stock_volume_cm3, material):
    """Convert stock volume to weight in kg."""
    density = DENSITY[material]
    weight_grams = stock_volume_cm3 * density
    return round(weight_grams / 1000.0, 2)  # kg
```

---

## Example

```
Bounding box: 120 × 40 × 30 mm
Turning: OFF → rectangular stock
Stock volume: (130 × 50 × 40) / 1000 = 260 cm³
Material: Steel (7.85 g/cm³)
Stock weight: 260 × 7.85 / 1000 = 2.04 kg
```

## Display to Engineer (Editable)

The dashboard shows the stock assumption and weight estimate, and allows edits:

```
System: Stock assumption: Rectangular block (Turning OFF)
    Estimated stock weight: 2.04 kg
    [Accept]  [Override]

Engineer: Override → 2.30 kg  (using nearest available plate size)
```

If overridden, the UI records it as an **Engineer override** (optionally with a short note).

For transparency, the UI can also display:

- Part volume (from Step 2)
- Estimated stock volume
- Material waste % (approx.) = $1 - \frac{\text{part volume}}{\text{stock volume}}$

## Output

- `estimated_stock_weight_kg` — rule-based estimate
- `stock_weight_final_kg` — engineer-confirmed value (equals estimate unless overridden)

`stock_weight_final_kg` feeds the Material Cost formula in Step 10.
